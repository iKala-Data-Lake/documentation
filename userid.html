# Visitor ID / User ID

# Visitor ID

By default, iKala Data Lake SDK of web/app will assign a unique Visitor ID to each device, and each Visitor ID will be counted as a unique user in the report.

So every time the user interact with web/app still will be recognized as the same user.

The Visitor ID is saved in browser cookie. If user use different browsers, browser will generate different Visitor ID. Incognito mode of browser can't save Visitor ID in cookie, so the user use browser with incognito mode also be recognized as new visitor every time.

App like iOS/Android generate Visitor ID and save it in App. If user doesn't delete app or delete app data, Visitor ID will exist permanently.

# User ID

When the user registers at your web/app. Your web/app have to generate an unique ID for the user, This ID can also be set as User ID in iKala Data Lake. Remember to use setUserId function to tell iKala Data Lake which user is using web/app after he/she login.

With User ID, even if the user uses web/app on a different device or browser, you can classify his behavior on the same user, which is not possible with Visitor ID.

# Set a user ID

When user login web/app, Your service will pass user id to front-end of web/app. Just use setUserId command to set User ID, so iKala Data Lake can recognized the user as same user even in different devices.

Please do not use sensitive personal information as User ID. ex: email.

If User ID is email. Hackers has opportunity to trace your user' other sensitive personal information by email.

You can set a user ID with the following code:

# Web - Javascript

ika.setUserId("USER_123");

# Android - Java

tracker.setUserId("USER_123");

# iOS - Swift

Tracker.shared.setUserId(userId: "USER_123")

After setting a User ID, all future hits will contain this User ID, and you can access it by querying for the User ID value in iKala Data Lake.
