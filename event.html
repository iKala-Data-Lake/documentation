# Event

Events are user interactions with content that can be measured independently from a web page or a screen load. Add product to cart, clicks, are all examples of actions you might want to measure as Events.

There are many defined events. Please use the event defined by iKala Data Lake to record user behavior.

Event hit list:

  • click
  • searchWord
  • searchFilter
  • addWish
  • removeWish
  • addCart
  • removeCart
  • viewitemList
  • viewItem
  • getCoupon
  • useCoupon

Following code shows example of sending event hit to iKala Data Lake:

# Web - Javascript

ika.addEvent('searchWord',
    {
        'word': 'tea',  // keyword which you searched
        'resultSize': 20,  // number of searched result
    }
);

# Android - Java

Tracker tracker = Tracker.getInstance(context);
Bundle bundle = new Bundle();
bundle.putString(EventParam.KEY_WORD, "tea"); // keyword which you searched
bundle.putInt(EventParam.KEY_RESULT_SIZE, 20);    // number of searched result
tracker.addEvent(EventName.EVENT_SEARCH_WORD, bundle);

# iOS - Swift

try? Tracker.shared.addEvent(name: .searchWord, params: [
    EventParam.word: "tea",
    EventParam.resultSize: 20
])

# Event hits list

# click

After click element on web/app, send this event.

Parameters

Key Value Type Required Description
which string No The name of element clicked by user.

# searchWord

Search products by keyword. SDK sends this event after server return searched result.

Parameters

Key Value Type Required Description
word string No The search string/keyword used.
resultSize integer No The number of search results.

# searchFilter

"searchFilter" event mean search products by many condition . Send this event When user clicked search button.

Parameters

Key Value Type Required Description
itemCateId string No The unique ID of item category.
payType string No Payment type which choosed by consumer. ex:visa.
shipType string No Shipping type which choosed by consumer. ex:fedex.
minPrice float No Minimum price of item which is acceptable by consumer.
maxPrice float No Maximum price of item which is acceptable by consumer.

# getCoupon

Coupon codes, also known as promo codes or discount codes. "getCoupon" event means user get coupon code from web/app.

Parameters

Key Value Type Required Description
couponId string No Coupon code for item which can be purchased.
couponType string No coupon name or description. ex:summer_discount,50_percent_off.
couponAmount float No discount amount.

# useCoupon

Send this event when user enter coupon code into web/app.

Parameters

Key Value Type Required Description
couponId string No Coupon code for item which can be purchased.
couponType string No coupon name or description. ex:summer_discount,50_percent_off.
couponAmount float No discount amount.

# confirmCartList

When user click checkout button at shopping cart page, send this event.

Parameters

Key Value Type Required Description
itemIdList string No List of item ID. It is a comma separated string to represent more than one item. ex: ITEMID_123,ITEMID_456,ITEMID_789.
skuList string No List of Stock Keeping Unit. It is a comma separated string to represent more than one item. ex: SKU-A-1,SKU-B-2,SKU-C-3.
quantity float No The quantity of a item.
currency string No The currency of a item.
price float No The price of a item.

# checkoutProcess

You can customized your checkout steps in checkout process. ex: choosing payment type, choosing shipping type, fill in credit card number, etc.

Send this event when users confirm their payment type or other decisions.

Parameters

Key Value Type Required Description
step integer No step of checkout flow.
option string No option on step in checkout flow. ex: use visa as payment, or use fedex as shipping.
